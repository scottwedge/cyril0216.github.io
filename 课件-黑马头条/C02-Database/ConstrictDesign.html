2.2.约束设计

  • 约束设计

约束设计

  • 关系型数据库通过 设置约束 来 限制加入表的数据的类型。
  • 可以在创建表时设置约束（通过 CREATE TABLE 语句），或者在表创建之后也可以（通过 ALTER TABLE 语句）。
  • Mysql中提供了以下五种约束:
    • UNIQUE 唯一约束
    • NOT NULL 非空约束
    • DEFAULT 默认值约束
    • PRIMARY KEY 主键约束
    • FOREIGN KEY 外键约束

代码示例

  • 建表时添加约束
create table t_user(
  id bigint not null auto_increment,  # 非空约束
  mobile char(11),
  type tinyint default 0,  # 默认值约束
  unique (mobile),  # 唯一约束
  primary key (id),  # 主键约束
)

create table t_address(
  id bigint not null auto_increment,
  detail varchar(40),
  userid bigint,
  foreign key (userid) references t_user(id)  on delete cascade | on delete set null  # 外键约束
)
  • 查看表的约束情况
show create table t_user;
  • 建表以后设置约束
# 唯一约束
添加:alter table table_name add unique 约束名（字段）
删除:alter table table_name drop key 约束名

# 非空约束
添加:alter  table table_name modify 列名 数据类型  not null 
删除:alter table table_name modify 列名 数据类型 null

# 默认值约束
添加:alter table table_name alter 列名  set default '值'
删除:alter table table_name alter 列名  drop default

# 主键约束
添加:alter table  table_name add primary key (字段)
删除:alter table table_name drop primary key

# 自动增长
添加:alter table table_name  modify 列名 int auto_increment
删除:alter table table_name modify 列名 int 

# 外键约束
添加:alter table table_name add constraint 约束名 foreign key(外键列) references 主键表（主键列）
删除:alter table table_name drop foreign key 约束名

注意点

  • NOT NULL
    • 最常见的使用字段为主键
  • DEFAULT
    • 一般 在 状态/类型 以及 日期/时间 字段中经常使用
    • 如:
      CREATE TABLE `mytest` (
`text` varchar(255) DEFAULT '' COMMENT '内容',
`create_time` timestamp NOT NULL DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',  # 生成记录时, 字段值根据当前时间自动生成
`update_time` timestamp NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间'  # 记录时, 字段值根据当前时间自动更新
) ENGINE=InnoDB DEFAULT CHARSET=utf8;
  • 字段的值尽量不要设置为NULL
    • 缺点
      • 可空列会占用更多空间
      • 聚合查询 / not in / != 语句, 值为NULL的数据不会被统计
      • 索引不存储null值, 查询时不会被统计
    • 解决办法
      • 给字段设置 默认值约束
      • 给字段设置 非空约束, 让SQL进行限制提醒
      • 拓展阅读: 不用Null的理由
  • UNIQUE
    • 优点:
      • 可以实现 消息幂等
      • 匹配唯一键的 查询效率极高 (仅次于主键)
    • 举例: 昵称, 手机号
  • 主键约束
    • NOT NULL + UNIQUE 的组合
  • 外键约束
    • 不影响查询, 但影响增删改效率
      • 删除主表数据时, 会到从表中查询一次关联数据 (如果是逻辑删除不受影响)
      • 更新主表的关联字段时, 会到从表中查询一次关联数据 (关联字段往往是主表主键, 一般也不受影响)
      • 向从表添加数据时, 会到主表中查询对应的关联数据是否存在, 不存在会报错; 存在则会给主表的该数据加锁, 防止从表数据还没有生成的情况下, 主表数据被修改/删除
    • 实际开发中, 一般 不建立外键约束 (物理外键), 只定义普通的字段记录主键(逻辑外键)
      • 仍然可以多表联查
      • 提高增删改效率
      • 有出错的风险, 由程序逻辑来控制

 

ORM与外键约束 (拓展)

  • 使用django-orm建表时, 定义的外键字段默认会创建外键约束, 只要设置参数 db_constraint=False 则不会生成外键约束, 只生成普通外键字段, 如:

    class BookInfo(models.Model):
  """图书模型类"""
  btitle = models.CharField(max_length=20, verbose_name='标题')
  bpub_date = models.DateField(verbose_name='出版日期')

class HeroInfo(models.Model):
  """英雄模型类"""  
  hname = models.CharField(max_length=20, verbose_name='名称')
  hbook = models.ForeignKey('BookInfo', verbose_name='所属图书', db_constraint=False)  # 取消外键约束

  • 使用sqlalchemy建表时, 定义的外键字段如果设置了外键参数db.ForeignKey, 也会生成外键约束, 但是 sqlalchemy 没有提供取消外键约束的相关语法, 解决办法为:

    • 使用sqlalchemy建表时不设置外键参数/使用SQL建表, 项目中数据操作时如果使用关系属性再添加外键参数(不参与建表不会生成约束), 如:
    """用于建表的模型类文件"""
class User(db.Model):
    id = db.Column(db.Integer, primary_key=True)
    name = db.Column(db.String(20))

class Address(db.Model):
    id = db.Column(db.Integer, primary_key=True)
    detail = db.Column(db.String(20))
    user_id = db.Column(db.Integer)  # 定义外键

if __name__ == '__main__':
  # 创建所有继承自db.Model的表
  db.create_all()

    """用于数据操作的模型类文件"""
class User(db.Model):
    id = db.Column(db.Integer, primary_key=True)
    name = db.Column(db.String(20))
    addresses = db.relationship('Address')  # 1.定义关系属性

class Address(db.Model):
    id = db.Column(db.Integer, primary_key=True)
    detail = db.Column(db.String(20))
    user_id = db.Column(db.Integer, db.ForeignKey('t_user.id'))  # 2. 定义外键参数
