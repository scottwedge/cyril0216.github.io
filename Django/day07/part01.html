Introduction

Request 与 Response

1. Request 类

这一小节, 我们将介绍一种全新的类: Request 类.

# Django REST framework 这个框架中定义了一种新的对象: 

# 在 DRF 中: 
# 我们今天讨论的重点
request = Request()

# 在 Django 中:
# 我们以前接触到的
request = HttpRequest()

这个对象和我们以前接触的 request 对象不同:

  • 在 DRF 框架中 request 对象的创建者是 Request 类.

  • 而 Request 类的是对 Django 默认的 HttpRequest 类的封装.

  • 故:

    • Django 中: HttpRequest 类 ======> 原始的 request 对象
    • DRF框架中: Request 类 =========> 我们这里说的 request 对象

  • Request 类 和 HttpRequest 类有什么关联呢?

    • Request 类 是对 HttpRequest 类进行的封装.

    • Request 类对 HttpRequest 类进行了扩展. 例如: 添加了 parse 解析的功能.

        Django REST Framework 提供了Parser解析器，在接收到请求后会自动根据 Content-Type 指明的请求数据类型（如JSON、表单等）将请求数据进行 parse 解析，解析为类字典对象保存到 request 对象中

      前端发送 json 数据 ==> parse自动解析 ===> request对象的数据

      以上做法的好处:

      无论前端发送的哪种格式的数据，我们都可以以统一的方式解析读取数据

            request.POST    # 只能处理数据， 在表单数据传输时使用。
      request.data    # 处理任意数据，在POST, PUT, PATCH请求下都起作用

常用属性

我们通过 Request 类,可以获取对应的 request 对象

拿到这个对象之后, 这个对象有一些相关的属性, 我们可以调用:

1）.data

data 属性作用: 主要是为了获取 parse 解析后请求体的数据

使用方式: 

# 获取自动解析后 请求体 的数据
# 这里一定要知道, 获取的是请求体的数据, 而不是获取的url里面的数据
data = request.data

request.data 返回解析之后的请求体数据

故:

​ 这里的 data 类似于我们调用: request.POST和request.FILES获取的数据

特点:

​ 都是通过请求体获取的数据

但提供如下特性：

  • 包含了解析之后的文件和非文件数据
  • 包含了对POST、PUT、PATCH请求方式解析后的数据
  • 利用了REST framework的parsers解析器，不仅支持表单类型数据，也支持JSON数据

2）.query_params

query_params 属性的作用: 主要是为了获取 parse 解析后 URL 中的数据

使用方式: 

# 获取自动解析后 url 的数据
# 和上面正好相反:
data = request.query_params

这里对应着我们以前调用的 request.GET 获取的数据.

request.query_params与Django标准的request.GET相同，只是更换了名称而已。

2. Response

使用 Response 对象时只需要传入数据即可，Response 会自动返回客户端需要的类型

和上面的 Request类 类似, 这里的 Response类 也是对 Django 中的 HttpReponse类 进行的封装

父类是: SimpleTemplateResponse

使用方式: 

from rest_framework.response import Response

def xxx():
    ...
    return Response('响应的内容', status=状态码, ...)

自动转换类型的奥秘:

Django REST framework 提供了 Render 渲染器

用来根据请求头中的 Accept（接收数据类型声明）来自动转换响应数据到对应格式。

DRF 框架提供了 Response 类: 

该类能够根据请求头的 Accept 字段自动进行 render 渲染

默认的渲染方式: 

REST_FRAMEWORK = {
     # 默认响应渲染类
    'DEFAULT_RENDERER_CLASSES': ( 
         # json方式渲染
        'rest_framework.renderers.JSONRenderer', 
         # 浏览器方式渲染
        'rest_framework.renderers.BrowsableAPIRenderer', 
    )
}

如果前端请求中未进行Accept声明，则会采用默认方式处理响应数据

我们也可以通过配置来修改默认响应格式

Response 的构成

Response(data, status=None, template_name=None, headers=None, content_type=None)

参数解读:

  • data: 只需传递 python 中常见的数据类型即可, 自动转换

data 不能是复杂结构的数据，如 Django 的模型类对象，对于这样的数据我们可以使用 Serializer 序列化器序列化处理后（转为了 Python 字典类型）再传递给 data 参数。

  • status : 状态码，默认200；我们这里可以使用下面定义好的状态码.
  • template_name: 模板名称，如果使用 HTMLRenderer 时需指明；
  • headers: 用于存放响应头信息的字典；
  • content_type: 响应数据的 Content-Type，通常此参数无需传递，REST framework会根据前端所需类型数据来设置该参数。

常用属性：

我们可以在这里查看一下 response 对象常用的一些属性:

1）.data

传给 response 对象的序列化后，但尚未 render 处理的数据

2）.status_code

状态码的数字

3）.content

经过 render 处理后的响应数据

3. 状态码

Django REST framewrok 在 rest_framework.status 模块中提供了常用状态码常量

Response(data, status=None, template_name=None, headers=None, content_type=None)
