注册页面---判断用户名存在的接口

学习目标:

  • 定义接口, 判断用户名是否已经注册

作用:

  • 帮助用户了解, 他所写的用户是否已经存在, 如果存在, 需更换一个. 如果不存在, 可以使用.

接口设计：

请求方式： 

# 确定请求方式为 GET 
# 需要把用户名传过来, 所以需要有参数, 我们给参数命名为: username

GET      usernames/(?P<username>\w{5,20})/count/

请求参数： 路径参数

参数 类型 是否必传 说明
username str 是 用户名

返回数据： JSON

{
    "username": "itcast",
    "count": "1"
}
返回值 类型 是否必须 说明
username str 是 用户名
count int 是 数量

视图原型：

# TODO 3.5 给注册页面用户名是否存在的接口配置 url

    """
    1.获取指定用户名数量
    2.拼接为字典  其中包含有: 用户名 和 数量
    3.返回
    """
    pass

代码实现

在 users / views.py 中定义视图

# TODO 3.6 创建验证用户名的接口
class UsernameCountView(APIView):
    """
    用户名数量
    """
    def get(self, request, username):
        """
        获取指定用户名数量, 并返回
        """
        count = User.objects.filter(username=username).count()

        data = {
            'username': username,
            'count': count
        }

        return Response(data)

前端实现

在 js / register.js 中修改

    // 检查用户名
    check_username: function (){
            var len = this.username.length;
            if(len<5||len>20) {
                this.error_name_message = '请输入5-20个字符的用户名';
                this.error_name = true;
            } else {
                this.error_name = false;
            }
            // TODO 3.6 修改前端代码, 增加功能

                        responseType: 'json'
