Introduction

CORS 跨域

学习目标:

  • 了解 cors 跨域
  • 使用扩展添加 cors 跨域

作用:

  • 能够使前端跨域获取资源, 而不会受限制

浏览器的 同源 策略:

协议、主机ip 和 端口port 都相同的两个地址是同源地址，否则是非同源地址

非同源:

当发起请求的页面地址和被请求的地址不是同源，那么这个请求就是跨域请求

总结:

  • 在发起请求时，如果浏览器发现请求是跨域请求，会在请求的报文头中，自动添加如下信息:

    Origin: 源请求IP地址 

# 例如：
Origin: http://www.meiduo.site:8080

  • 在服务器返回的响应中，如果响应头中包含如下信息, 则代表可以进行跨域请求

    Access-Control-Allow-Origin: 源请求IP地址

# 例如：
Access-Control-Allow-Origin: http://www.meiduo.site:8080

    浏览器会自动判断被访问服务器是否支持跨域

    如果认为支持, 会添加 origin 字段

    认为不支持，浏览器会将请求直接驳回。

Django-cors-headers 扩展类

在 Django 中, 我们使用 Django-cors-headers 扩展类来解决跨域问题.

学习目标:

  • django-cors-headers 的安装和配置

作用:

  • 解决 django 开发中遇到的跨域访问资源问题

我们为前端和后端分别设置了两个不同的域名

位置 域名
前端 www.meiduo.site
后端 api.meiduo.site

现在，前端与后端分处不同的域名，我们需要为后端添加跨域访问的支持。

我们使用CORS来解决后端对跨域访问的支持

使用django-cors-headers扩展:

参考文档https://github.com/ottoyiu/django-cors-headers/

安装

# 安装 django-cors-headers 解决跨域问题:
pip install django-cors-headers

添加应用


INSTALLED_APPS = (
    ...
    # TODO 2.6 添加 django-cors-headers 的配置内容, 使其可以进行cors跨域
    'corsheaders',
    ...
)

中间层设置

MIDDLEWARE = [
    # TODO 2.7 添加 django-cors-headers 的配置内容, 使其可以进行cors跨域
    'corsheaders.middleware.CorsMiddleware',
    ...
]

添加白名单

# TODO 2.8 添加 django-cors-headers 的白名单, 使白名单中的 host 可以进行跨域请求
CORS_ORIGIN_WHITELIST = (
    # 白名单:
    '127.0.0.1:8080',
    'localhost:8080',
    'www.meiduo.site:8080',
)
# 允许白名单中的 host 跨域请求时携带 cookie
CORS_ALLOW_CREDENTIALS = True
  • 凡是出现在白名单中的域名，都可以访问后端接口
  • CORS_ALLOW_CREDENTIALS 指明在跨域访问中，后端是否支持对cookie的操作。

代码优化:

针对我们之前的代码, 我们有两个地方可以进行对应的优化:

  • 用户点击获取验证码按钮之后, 最好60s内不让用户再次获取
  • 我们存储到 redis 数据库中的两部操作可以使用 管道 简化.

学习目标:

  • 完善我们之前添加的代码

作用:

  • 限制用户发送请求, 获取短信验证码的次数
  • 优化 redis 的读写操作, 减少读写次数

优化一:

用户点击 获取验证码 按钮之后, 最好规定一个事件, 在这个时间之内, 不允许用户再次获取, 这样可以减少服务器端的压力:

所以我们可以在用户第一次获取验证码时, 做一个记录, 记录到 redis 之中. 下次用户在进行请求时, 我们先从 redis 之中获取这个手机号, 查看是否有对应的记录, 如果有记录, 则直接拒绝. 否则生成验证码.

  • 代码如下: 
# TODO 2.9 我们也可以对当前代码进行相应的优化, 使其能够判断是否发送短信过于频繁
class SMSCodeView(APIView):
    def get(self, request, mobile):
        # 获取 redis 第三个存储区域的链接:
        redis_conn = redis_conn = get_redis_connection('verify_codes')
        # 从 redis 中根据 mobile 获取记录
        send_flag = redis_conn.get('send_flag_%s' % mobile)
        # 判断是否有这个记录, 如果有, 直接返回.
        if send_flag:
            return Response({'message':'发送短信过于频繁'}, status=status.HTTP_400_BAD_REQUEST)

        # 生成验证码
        sms_code = '%06d' % random.randint(0, 999999)

        # 如果没有这个记录, 则在 redis 中保存记录.方便下次获取.
        pl.setex('send_flag_%s' % mobile, constants.SEND_SMS_CODE_INTERVAL, 1)

constants.py 文件如下: 

# 发送短信时间间隔限制: s
SEND_SMS_CODE_INTERVAL = 60

优化二:

从上面的代码可知, 我们有两次存入 redis 数据库的操作.

所以我们可以针对这两次操作进行优化:

  • 使用管道:

    redis管道: 可以向管道中添加多个要执行redis命令, 然后一次性执行

代码如下:

   # TODO 2.10 使用管道

    # 这两步可以省略:
    # redis_conn.setex('sms_%s' % mobile, constants.SMS_CODE_REDIS_EXPIRES, sms_code)
    # redis_conn.setex('send_flag_%s', mobile, constants.SEND_SMS_CODE_INTERVAL, 1)

    # 上面两部可以省略为一步:
    # redis管道: 可以向管道中添加多个要执行redis命令, 然后一次性执行
    # 生成管道:
    pl = redis_conn.pipeline()
    # 在管道中放入操作:
    pl.setex('sms_%s' % mobile, constants.SMS_CODE_REDIS_EXPIRES, sms_code)
    pl.setex('send_flag_%s' % mobile, constants.SEND_SMS_CODE_INTERVAL, 1)
    # 执行管道:
    pl.execute()
