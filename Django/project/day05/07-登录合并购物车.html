登录合并购物车

在用户登录时，将 cookie 中的购物车数据合并到 redis 中，并清除 cookie 中的购物车数据

普通登录和QQ登录都要合并，所以将合并逻辑放到公共函数里实现

例如:

假如登录之前的数据如下所示: 

# 登录之前 cookie 中的购物车数据: 
{
    10: {
        'count':1,
        'selected':False
    },
    20: {
        'count':2,
        'selected':True
    }
}


# 登录之前 redis 中的购物车数据如下: 
cart_2: {
    '10': '2',
    '20': '1',
    '30': '1'
}

cart_selected_2: ('10', '30')

合并:

我们选择的合并方案是: 以 cookie 中的购物车数据为准, 如果出现冲突, 则使用 cookie 中的数据 覆盖 redis 中的原有记录.

如果没有冲突, 则都保留

# 登录合并之后 redis 中的购物车数据如下: 
cart_2: {
    '10': '1',
    '20': '2 ',
    '30': '1'
}

cart_selected_2: ('30', '20')

在 carts / utils.py 中创建 merge_cart_cookie_to_redis 方法

import pickle
import base64
from django_redis import get_redis_connection


def merge_cart_cookie_to_redis(request, response, user):
    """
    合并请求用户的购物车数据，将未登录保存在cookie里的保存到redis中
    遇到cookie与redis中出现相同的商品时以cookie数据为主，覆盖redis中的数据
    :param request: 用户的请求对象
    :param user: 当前登录的用户
    :param response: 响应对象，用于清楚购物车cookie
    :return:
    """
    # 获取cookie中的购物车
    cookie_cart = request.COOKIES.get('cart')
    if not cookie_cart:
        return response

    # 解析cookie购物车数据
    cookie_cart = pickle.loads(base64.b64decode(cookie_cart.encode()))

    # 用于保存向redis购物车商品数量hash添加数据的字典
    cart = {}

    # 保存cookie中被勾选商品的sku_id,在合并时准备将这些id加入redis的set中:
    redis_cart_selected_add = []

    # 保存cookie中没有被勾选商品的sku_id,在合并时准备将这些id从redis的set中移出:
    redis_cart_selected_remove = []

    # 合并cookie购物车与redis购物车，保存到cart字典中
    for sku_id, count_selected in cookie_cart.items():
        # 处理商品数量    
        # 格式变形: 将cookie中存储的数据变为: {'sku_id': 'count', ...} 的形式:
        cart[sku_id] = count_selected['count']

        # 判断cookie中的字段是否勾选: 
        if count_selected['selected']:
            # 如果勾选了, 则保存到 redis_cart_selected_add 中
            redis_cart_selected_add.append(sku_id)
        else:
            # 如果没有勾选, 则保存到 redis_cart_selected_remove
            redis_cart_selected_remove.append(sku_id)

    # 判断 cart 中是否有值: 
    if cart:
        # 获取数据库连接: 
        redis_conn = get_redis_connection('cart')
        pl = redis_conn.pipeline()

        # 合并购物车中商品的sku_id和对应数量count:
        # hmset: 能够同时将多个属性值, 设置进去.如果已经存在会进行覆盖. 
        pl.hmset('cart_%s' % user.id, cart)

        # 合并购物车中商品勾选状态: 
        # 判断是否存在要添加的:
        if redis_cart_selected_add:
            # 如果有, 则加入: 
            pl.sadd('cart_selected_%s' % user.id, *redis_cart_selected_add)

        # 如果有要移出的:
        if redis_cart_selected_remove:
            # 则移出: 
            pl.srem('cart_selected_%s' % user.id, *redis_cart_selected_remove)

        pl.execute()

    # 清除cookie中的购物车数据: 删掉cookie
    response.delete_cookie('cart')

    return response

修改登录视图

rest_framework_jwt 提供的 obtain_jwt_token 视图

实际从rest_framework_jwt.views.ObtainJSONWebToken 类视图而来

我们可以重写此类视图里的 post 方法来添加合并逻辑

在 users / views.py 中, 我们定义一个类,重写 post 函数:

from rest_framework_jwt.views import ObtainJSONWebToken

# 我们重新定义一个新的类, 让该类继承自 ObtainJSONWebToken
# 所以我们自定义的类中就有了 post 函数, 重写 post 函数
# 在反序列化之后, 进行合并操作: 
class UserAuthorizeView(ObtainJSONWebToken):
    """
    用户认证
    """
    def post(self, request, *args, **kwargs):
        # 这些代码都是原来的代码: 
        serializer = self.get_serializer(data=request.data)
        if serializer.is_valid():
            user = serializer.object.get('user') or request.user
            token = serializer.object.get('token')
            response_data = jwt_response_payload_handler(token, user, request)
            response = Response(response_data)
            if api_settings.JWT_AUTH_COOKIE:
                expiration = (datetime.utcnow() +
                              api_settings.JWT_EXPIRATION_DELTA)
                response.set_cookie(api_settings.JWT_AUTH_COOKIE,
                                    token,
                                    expires=expiration,
                                    httponly=True)

            # 补充购物车记录合并的过程: 调用合并购物车记录函数:
            # 导入: 
            from carts.utils import merge_cart_cookie_to_redis
            # 调用我们自己封装的合并函数: 
            # 这里传入的参数 request, user, response
            # 都是从上面获取的: 
            merge_cart_cookie_to_redis(request, response, user)

            return response

        return Response(serializer.errors, status=status.HTTP_400_BAD_REQUEST)
