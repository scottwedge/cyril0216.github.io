Introduction

验证邮箱链接

学习目标:

  • 用户收到邮件后, 点击链接进行验证, 则会调用该接口
  • 实现该接口的功能, 使用户能够进行验证

作用:

  • 提供验证接口, 验证用户发来的是否正确

当用户点击邮箱里的链接时，进入到 success_verify_email.html 页面

注意:

在该页面中，我们将请求网址中用于验证的 token 发送给后端接口，由后端接口判断 token 的有效性，如果 token 有效，则修改邮箱的验证状态，并将处理结果返回给前端展示给用户。

接口设计：

该接口的目的是为了修改 User 表中 email_active 字段, 使其达到已验证过得目的

请求方式：PUT /emails/verification/?token=xxx

注意:

我们把当前接口写在了 users 子应用中, 但是这里的 url 需要注意

是: emails/verifications, 而不是 /verifications:

url(r'^emails/verification/$', views.VerifyEmailView.as_view()),

请求参数： 查询字符串参数

参数 类型 是否必须 说明
token str 是 用于验证邮箱的token

返回数据： JSON

返回值 类型 是否必须 说明
message str 是 验证处理结果

在 users / views.py 中新建视图

# url(r'^emails/verification/$', views.VerifyEmailView.as_view()),
class VerifyEmailView(APIView):
    """
    邮箱验证
    """
    def put(self, request):
        # 获取token
        token = request.query_params.get('token')
        if not token:
            return Response({'message': '缺少token'}, status=status.HTTP_400_BAD_REQUEST)

        # 验证token
        user = User.check_verify_email_token(token)
        if user is None:
            return Response({'message': '链接信息无效'}, status=status.HTTP_400_BAD_REQUEST)
        else:
            user.email_active = True
            user.save()
            return Response({'message': 'OK'})

在 User 模型类中定义验证 token 的方法

from itsdangerous import TimedJSONWebSignatureSerializer
from django.conf import settings
from itsdangerous import BadData

class EmailSerializer(serializers.ModelSerializer):
    ...

    # 在序列化器中, 补充如下方法: 
    @staticmethod
    def check_verify_email_token(token):
        """
        检查验证邮件的token
        """
        serializer = TimedJSONWebSignatureSerializer(settings.SECRET_KEY, expires_in=7200)
        try:
            data = serializer.loads(token)
        except BadData:
            return None
        else:
            email = data.get('email')
            user_id = data.get('id')
            try:
                user = User.objects.get(id=user_id, email=email)
            except User.DoesNotExist:
                return None
            else:
                return user

# 邮箱验证路由:
